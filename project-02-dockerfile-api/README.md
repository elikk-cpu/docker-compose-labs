# Проект 02. Контейнеризация простого API

**Сложность:** начальная  
**Оценка времени:** 5–7 часов

## Ситуация

Разработчики передали Flask API. Нужно подготовить воспроизводимый image для
тестового сервера.

## Задание

1. Напиши `Dockerfile`.
2. Напиши `.dockerignore`.
3. Используй versioned base image.
4. Передай `APP_VERSION` через build argument и сохрани его как runtime `ENV`.
5. Организуй слои так, чтобы изменение `app.py` не переустанавливало зависимости.
6. Запусти API через production WSGI server, уже указанный в requirements.
7. Опубликуй API только на localhost хоста.
8. Проверь `/healthz` и `/api/info`.
9. Выполни две последовательные сборки и объясни cache hits.
10. Воспроизведи ошибку прослушивания `127.0.0.1` и неверный CMD.

## Ограничения

- Не используй `latest`.
- Не копируй `.env`, `.git` или virtualenv в image.
- Не исправляй код внутри запущенного контейнера.
- Не оптимизируй безопасность раньше проекта 03: здесь важны основы Dockerfile.

## Ты создаёшь

`Dockerfile`, `.dockerignore`, `RESULT.md`.

## После выполнения мы научились

Работать с Dockerfile, build context, слоями, cache invalidation, ARG/ENV,
EXPOSE и exec-формой запуска.

## Где пригодится

Контейнеризация сервисов разработчиков и ускорение сборок в CI.
